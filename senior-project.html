Senior project

Description

For my senior project I wanted to create a messaging app that takes security as its top priority. The goal was full encryption from end to end, with temporary messages that only persisted long enough to be viewed by the user.

Ultimately not all of our goals were met. However, we were able to successfully create a messaging application that encrypted its messages. The only missing piece was having messages leave after being viewed.

Background

When I first started this jourey for my senior project I reached out to my co-workers and fellow developers to get some ideas for a good application. There were a few good ones, but the one that stood out to me was the suggestion of a completely encrypted messaging application. We talked in length of what we would want the project to be and the many interesting functionalities that it could hold. Eventually when I started writing up the project and the expectations that I held for it, the list was extensive and far more than I could handle in the given time frame. After some decision making I was able to dwindle it down to its core funcitonality as a proof of concept with future changes that are left out of scope for the time being.

The final product is something that I am proud of for the time given to me. I was able to create an application that a user can register to, login, search for other users, and send/recieve messages. This on its own proved to have some problems as the projected started coming together. The original design did not include any form of saved messages or users for that matter. With the lack of a database the application was almost useless. Once a database was integrated problems began as I had to find a way to properly let my JavaScript based Vue project to connect to my database. After consulting with my coworkers I added an API to my project using PHP.
